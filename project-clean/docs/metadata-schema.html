DOCiD Metadata Properties

Comprehensive metadata framework for publication and document identifier management following international standards

Metadata Schema Overview

DOCiD implements a comprehensive metadata schema designed for "Publication and Citation of Research Data and Other Research Outputs". Based on DataCite Metadata Schema 4.6, it provides a robust framework for documenting research artifacts with precise, standardized information, while extending capabilities for African research contexts and Indigenous knowledge systems.

📋 Schema Characteristics

  • 20 core properties following DataCite standards
  • Extensive controlled vocabulary lists for field standardization
  • Hierarchical metadata structures with Publications model
  • JSON-based flexible metadata extensions
  • Multi-identifier support (DOI, Handle, DocID, CSTR)

🔑 Core Design Principles

  • Interoperability: Full DataCite 4.6 compliance
  • Extensibility: JSON metadata fields for custom properties
  • Cultural Sensitivity: Local Contexts and Indigenous protocols
  • Quality Assurance: Comprehensive validation and constraints
  • African Context: Multilingual and regional adaptations

DOCiD Publication Model

The core Publications model implements the metadata schema with extensive relational connections:

Publications Model Structure

class Publications(db.Model):
    # Core Identifiers
    id = db.Column(db.Integer, primary_key=True)
    docid = db.Column(db.String(50), unique=True)  # Property 1: Identifier
    doi = db.Column(db.String(100), unique=True)   # Property 1: Identifier
    handle = db.Column(db.String(200))             # Property 1: Identifier
    
    # Required Properties
    title = db.Column(db.Text, nullable=False)      # Property 3: Title
    resource_type_id = db.Column(db.Integer, db.ForeignKey('resource_types.id'))  # Property 10
    publication_type_id = db.Column(db.Integer, db.ForeignKey('publication_types.id'))
    
    # Descriptive Properties
    abstract = db.Column(db.Text)                   # Property 17: Description
    description = db.Column(db.Text)               # Property 17: Description
    language = db.Column(db.String(10))            # Property 9: Language
    
    # Technical Properties
    version = db.Column(db.String(50))             # Property 15: Version
    status = db.Column(db.String(20), default='draft')
    
    # Dates (Property 8)
    created_at = db.Column(db.DateTime, default=datetime.utcnow)
    updated_at = db.Column(db.DateTime, onupdate=datetime.utcnow)
    publication_date = db.Column(db.DateTime)      # Property 5: Publication Year
    
    # JSON Metadata Fields
    metadata = db.Column(db.JSON)                  # Extended metadata
    subjects = db.Column(db.JSON)                  # Property 6: Subject
    keywords = db.Column(db.JSON)                  # Property 6: Subject
    geo_location = db.Column(db.JSON)              # Property 18: Geo Location
    file_metadata = db.Column(db.JSON)             # Property 13: Size, 14: Format
    rights = db.Column(db.JSON)                    # Property 16: Rights
    
    # Relationships
    user_id = db.Column(db.Integer, db.ForeignKey('user_accounts.user_id'))
    
    # Related entities via foreign keys
    creators = db.relationship('PublicationCreators')           # Property 2: Creator
    organizations = db.relationship('PublicationOrganization')  # Property 4: Publisher
    funders = db.relationship('PublicationFunders')            # Property 19: Funding Reference
    files = db.relationship('PublicationFiles')                # Property 14: Format
    identifiers = db.relationship('PublicationIdentifiers')     # Property 11: Alternate Identifier
    comments = db.relationship('PublicationComments')
    relations = db.relationship('PublicationRelations')         # Property 20: Related Item

Creator Management (Property 2)

The PublicationCreators model implements comprehensive creator/author management with role-based assignments:

PublicationCreators Model

class PublicationCreators(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    publication_id = db.Column(db.Integer, db.ForeignKey('publications.id'))
    
    # Creator Information
    creator_name = db.Column(db.String(255), nullable=False)
    given_name = db.Column(db.String(100))
    family_name = db.Column(db.String(100))
    
    # Identifiers
    orcid_id = db.Column(db.String(19))           # ORCID integration
    scopus_id = db.Column(db.String(50))
    researcher_id = db.Column(db.String(50))
    
    # Role and Affiliation
    role_id = db.Column(db.Integer, db.ForeignKey('creators_roles.id'))
    affiliation = db.Column(db.String(500))
    affiliation_identifier = db.Column(db.String(100))  # ROR ID
    
    # Order and metadata
    creator_order = db.Column(db.Integer)
    creator_metadata = db.Column(db.JSON)
    
    # Relationships
    role = db.relationship('CreatorsRoles')
    publication = db.relationship('Publications')

Funding Reference (Property 19)

The PublicationFunders model implements structured funding information management:

PublicationFunders Model

class PublicationFunders(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    publication_id = db.Column(db.Integer, db.ForeignKey('publications.id'))
    
    # Funder Information
    funder_name = db.Column(db.String(255), nullable=False)
    funder_name_local = db.Column(db.String(255))
    
    # Identifiers
    funder_id = db.Column(db.String(100))         # Crossref Funder ID
    doi = db.Column(db.String(100))               # Funder DOI
    ror_id = db.Column(db.String(50))             # ROR ID if applicable
    
    # Grant Information
    grant_number = db.Column(db.String(100))
    grant_title = db.Column(db.String(500))
    program_name = db.Column(db.String(255))
    
    # Financial Information
    award_amount = db.Column(db.Numeric(15, 2))
    currency = db.Column(db.String(3))            # ISO currency code
    
    # Classification
    funder_type_id = db.Column(db.Integer, db.ForeignKey('funder_types.id'))
    funding_scheme = db.Column(db.String(100))
    
    # Geographic and temporal scope
    funding_country = db.Column(db.String(100))
    funding_start_date = db.Column(db.Date)
    funding_end_date = db.Column(db.Date)
    
    # Additional metadata
    funding_metadata = db.Column(db.JSON)
    
    # Relationships
    funder_type = db.relationship('FunderTypes')
    publication = db.relationship('Publications')

Identifier Management (Properties 1, 11, 12)

DOCiD implements comprehensive identifier management across multiple schemes:

PublicationIdentifiers Model

class PublicationIdentifiers(db.Model):
    id = db.Column(db.Integer, primary_key=True)
    publication_id = db.Column(db.Integer, db.ForeignKey('publications.id'))
    
    # Identifier Information
    identifier_value = db.Column(db.String(255), nullable=False)
    identifier_type_id = db.Column(db.Integer, db.ForeignKey('publication_identifier_types.id'))
    
    # Relationship and validation
    relation_type = db.Column(db.String(50))      # Property 12: Related Identifier
    is_primary = db.Column(db.Boolean, default=False)
    is_active = db.Column(db.Boolean, default=True)
    
    # Metadata
    assigned_date = db.Column(db.DateTime, default=datetime.utcnow)
    assigned_by = db.Column(db.String(100))       # System or external service
    validation_status = db.Column(db.String(20))  # valid, invalid, pending
    
    # Additional context
    identifier_metadata = db.Column(db.JSON)
    
    # Relationships
    identifier_type = db.relationship('PublicationIdentifierTypes')
    publication = db.relationship('Publications')

Data Validation & Constraints

DOCiD implements comprehensive validation rules to ensure metadata quality and consistency:

✅ Format Validation

  • Email: Valid email format for user accounts
  • ORCID ID: Format xxxx-xxxx-xxxx-xxxx with checksum
  • DOI: Valid DOI format (10.xxxx/xxxxx)
  • ROR ID: Valid ROR identifier format
  • URL: Valid URL format for web resources
  • Handle: Valid Handle System format

🔗 Referential Integrity

  • Foreign Key Constraints: All relationships validated
  • Cascading Deletes: Dependent records management
  • Unique Constraints: Identifier uniqueness enforcement
  • Not Null Constraints: Required field validation

📊 Business Rules

  • Creator Minimum: At least one creator required
  • Title Validation: Non-empty title requirement
  • Date Consistency: Publication date validation
  • Role Validation: Valid creator roles only

🌐 External Validation

  • ORCID Verification: Real-time ORCID validation
  • ROR Validation: Organization identifier verification
  • DOI Resolution: DOI accessibility checking
  • Crossref Validation: External DOI metadata validation

Metadata Workflow

DOCiD implements a comprehensive metadata lifecycle workflow:

1️⃣ Creation Phase

  • Basic metadata entry
  • Required field validation
  • Initial identifier assignment (DocID)
  • Creator and organization association

2️⃣ Enrichment Phase

  • External service integration

  • ORCID and ROR resolution
  • Subject classification assignment

3️⃣ Validation Phase

  • Comprehensive metadata validation
  • External identifier verification
  • Quality score calculation
  • Completeness assessment

4️⃣ Publication Phase

  • DOI registration (if applicable)
  • Handle assignment via CORDRA
  • External platform synchronization
  • Public visibility activation
